# Ionnetix Technologies — Development Rules

## 1. Code Structure

* Component-based architecture
* Reusable UI components
* Clean folder structure

---

## 2. Naming Conventions

* Components: PascalCase
* Variables: camelCase
* Files: kebab-case

---

## 3. Performance Rules

* Optimize images
* Lazy load sections
* Avoid heavy libraries
* Keep bundle size minimal

---

## 4. SEO Rules

* Proper heading hierarchy (H1 → H3)
* Meta titles & descriptions
* Clean URLs
* Fast loading speed

---

## 5. Content Rules

* No generic statements
* Focus on problems & solutions
* Short, impactful sentences
* Clear CTAs

---

## 6. UI/UX Rules

* Consistent spacing
* Consistent button styles
* Clear visual hierarchy
* Avoid clutter

---

## 7. Forms & Contact

* Keep forms minimal (name, contact, message)
* Always include WhatsApp option
* Fast response expectation

---

## 8. Scalability Rules

* Design components for reuse
* Prepare for future pages (blog, case studies)
* Avoid hardcoding data

---

## 9. Deployment Rules

* Use Vercel for hosting
* Test on mobile + desktop
* Fix all UI issues before launch

---

## 10. Maintenance

* Regular updates
* Add real projects ASAP
* Improve based on user interaction
